Answers

1. What is the difference between a normal div and a grid container?

A normal div (block element) lays out its children vertically, one after another. A grid container (display: grid) changes the layout context of its direct children, allowing them to be placed into a 2D grid of rows and columns. While visually they may look initially similar (staking vertically), the grid container enables powerful alignment and track sizing capabilities that a normal block container does not have.

2. Why does the layout look the same even after applying Grid?

The layout looks the same because, by default, a CSS Grid has a single column. The grid items (the children) are placed into implicit rows, one per row. Since block-level elements also stack vertically, the visual starting point is identical until you explicitly define columns (e.g., using grid-template-columns).

3. Are grid items created automatically?

Yes, all direct children of the element with display: grid automatically become grid items. You do not need to add any special class or property to the children for them to be part of the grid.

4. Can Grid work without rows or columns?

Yes. If you don't define explicit rows or columns, the grid will create them implicitly as needed to hold the content. By default, it creates one column and new rows for each item.

5. What do you think Grid needs to show visible changes?

To see a "grid" structure (items next to each other), you generally need to define grid-template-columns (e.g., grid-template-columns: 1fr 1fr 1fr;) or use grid-auto-flow: column;. This tells the browser to place items into multiple columns instead of just one vertical stack.
